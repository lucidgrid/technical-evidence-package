# Example: AI-Only Project TEP

This example shows a conformant Technical Evidence Package (TEP) for a production where all assets were generated entirely by AI tools, with no physical camera footage and no VES production context.

## What this example shows

- Two assets: one image (ComfyUI) and one video (Runway)
- No `fdl_link` and no `production_context` block — correct for AI-only projects
- `ves_taxonomy_enabled` omitted — correct for non-VES productions
- HITL evidence mix: Observed Truth (prompt captured from DOM), Contextual Truth (iteration inferred from session), Declared Truth (output curation declared by user)
- `evidence_completeness_score`: amber — reflects that one asset has only Declared Truth tags

## Package structure

```
ai_only_project/
├── README.md                         ← this file
├── MANIFEST.json                     ← package index + integrity contract
├── cryptographic_proof.json          ← asset hashes + OpenTimestamps anchor
├── attestation.json                  ← signed project-level declaration
├── indemnification_disclosure.json   ← dual-status vendor terms record
└── assets/
    └── asset_001/
        └── metadata.json             ← per-asset HITL evidence record
```

`executive_summary.pdf` is not included in this example — it is generated by LucidGrid at export time and is not a hand-authored file.

`workflow.json` is not included — this example uses Runway (browser capture), not ComfyUI. `workflow.json` is only required when ComfyUI is used.

## Reading this example

Start with `MANIFEST.json` to understand the package structure and integrity claims. Then read `assets/asset_001/metadata.json` to see how HITL evidence is recorded at the asset level. Then read `attestation.json` to see how the creative director signs off on the project. Finally, read `indemnification_disclosure.json` to see how dual-status vendor terms are recorded.
